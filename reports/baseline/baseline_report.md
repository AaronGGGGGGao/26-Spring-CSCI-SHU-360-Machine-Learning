# CSI500 XGBoost Baseline Report

## 1. Scope

This document records the current baseline method and its held-out results for the CSI500 stock selection competition. It is not the final submission report. Its purpose is to give a clean starting point for later model comparisons and to satisfy the project requirement that methods, experiments, and results be documented in a reproducible way.

## 2. Factors

The baseline uses only daily price and trading activity data from [prices.parquet](/Users/codes/projects/ml_competition/data/prices.parquet). No external data, fundamentals, industry labels, or news signals are included.

The feature set is implemented in [baseline/features.py](/Users/codes/projects/ml_competition/baseline/features.py) and includes:

- Short- and medium-horizon returns: `ret_1d`, `ret_5d`, `ret_10d`, `ret_20d`, `ret_60d`
- Volatility: `vol_20d`
- Volume anomaly: `volume_z_20d`
- Turnover level: `turnover_ma_20d`
- Trend / moving-average gap: `close_over_ma20`, `close_over_ma60`
- Momentum oscillator: `rsi_14`
- Cross-sectional daily ranks: `ret_5d_rank`, `ret_20d_rank`, `vol_20d_rank`

The design choice is conservative. These are standard technical signals that can be computed from the provided market data without introducing licensing issues or leakage from external sources.

## 3. Model

### 3.1 Prediction target

The baseline predicts a 5-trading-day forward return:

```python
df[TARGET_COLUMN] = close.shift(-FORWARD_HORIZON) / close - 1.0
```

With the current baseline settings:

- `TARGET_COLUMN = "target_5d"`
- `FORWARD_HORIZON = 5`

So each training sample is a `(date, stock_code)` pair with:

- input: the feature vector on date `t`
- label: the stock's forward 5-day return from `t` to `t+5`

### 3.2 Learning algorithm

The baseline model is an `XGBRegressor`, implemented in [baseline/baseline_xgboost.py](/Users/codes/projects/ml_competition/baseline/baseline_xgboost.py). Main settings:

- `n_estimators = 400`
- `max_depth = 5`
- `learning_rate = 0.05`
- `subsample = 0.8`
- `colsample_bytree = 0.8`
- `min_child_weight = 10`
- `reg_lambda = 1.0`
- `tree_method = "hist"`
- `early_stopping_rounds = 30`

### 3.3 Portfolio construction

The model does not output weights directly. It first predicts a cross-sectional score for each stock on the decision date, then converts scores into a long-only portfolio:

1. Rank all stocks by predicted score.
2. Select the top `K = 50` names.
3. Assign rank-based weights.
4. Cap any single weight at `10%`.
5. Redistribute excess weight to uncapped names until the portfolio satisfies the competition constraints.

This construction is implemented in `build_portfolio(...)` in [baseline/baseline_xgboost.py](/Users/codes/projects/ml_competition/baseline/baseline_xgboost.py).

## 4. Data Split and Leakage Control

### 4.1 Baseline development split

For the standard baseline run on the current data snapshot:

- raw data range: `2025-01-02` to `2026-04-24`
- usable training samples begin later because rolling features require history
- training rows: `117,371`
- validation rows: `4,985`

The baseline uses:

- training set up to `2026-03-26`
- 5-trading-day embargo
- validation set from `2026-04-03` to `2026-04-17`

The embargo exists because the target is a forward 5-day return. Without a gap, training labels near the split boundary would use future prices that overlap with the validation period.

This script uses a stricter split:


## 6. Analysis

### 6.1 What worked

- The baseline produces a legal portfolio and can be rerun end to end.
- The held-out test rank IC is positive (`0.1832`), which suggests the model captures some cross-sectional ordering signal.
- The held-out test excess return is also positive (`+0.916%` average over the test dates), so the portfolio construction is at least directionally aligned with the competition objective.
- The split uses explicit embargo windows, which is methodologically important because the label is forward-looking.

### 6.2 What did not work well

- The self-test validation rank IC is negative (`-0.0653`), which means the signal is unstable across neighboring windows.
- The validation and test results vary materially across short windows, confirming that this baseline is noisy and should not be treated as robust.
- The model uses only technical features from price data. It ignores fundamentals, industry effects, and market context, so it is likely under-specified.
- The baseline target is always 5-day forward return, while the first live evaluation window is only 3 trading days. That mismatch may reduce alignment with competition scoring.

### 6.3 Why this matters

The self-test requirement is not optional. The course explicitly requires a `training / validation / test` split and asks that the reported test performance be compared against the provided baseline. The current baseline plus [baseline/self_test_xgboost.py](/Users/codes/projects/ml_competition/baseline/self_test_xgboost.py) satisfies the structure of that requirement, but it is still only the reference model. To earn strong self-test credit, a later model should:

1. preserve clean time-based splitting with embargo
2. report held-out test performance
3. exceed this baseline's test results on the same split
